  Key CUDA-GDB Commands for Thread Control:

  # Focus on specific block and thread
  cuda kernel <kernel_id> block <bx,by,bz> thread <tx,ty,tz>

  # Examples:
  cuda kernel 0 block 0,0,0 thread 0,0,0   # Focus on first thread of first block
  cuda kernel 0 block 1,0,0 thread 15,0,0  # Block (1,0,0), Thread (15,0,0)

  # Show current focus
  cuda kernel block thread

  # List all kernels
  info cuda kernels

  # Show all blocks in current kernel
  info cuda blocks

  # Show all threads in current block  
  info cuda threads

  # Show warps
  info cuda warps

  # Switch to specific warp
  cuda warp <lane_mask>

  # Show current SM and warp
  info cuda sm
  info cuda warp

  Practical Debugging Example:

  (cuda-gdb) break vectorAdd2D        # Set breakpoint at kernel
  (cuda-gdb) run                      # Run program
  (cuda-gdb) info cuda kernels        # List running kernels
  (cuda-gdb) info cuda blocks         # Show active blocks

  # Focus on specific thread
  (cuda-gdb) cuda kernel 0 block 2,1,0 thread 5,3,0

  # Print thread indices
  (cuda-gdb) print threadIdx
  (cuda-gdb) print blockIdx
  (cuda-gdb) print blockDim

  # Step through only current thread
  (cuda-gdb) set cuda single-stepping on
  (cuda-gdb) next

  # Print variables for current thread
  (cuda-gdb) print x
  (cuda-gdb) print i

  Filtering/Limiting Threads:

  # Set conditional breakpoint for specific thread
  (cuda-gdb) break vectorAdd2D if blockIdx.x==0 && threadIdx.x==0

  # Break only when accessing specific memory index
  (cuda-gdb) break vectorAdd2D if i==100

  # Set scheduler to single-step one warp at a time
  (cuda-gdb) set cuda scheduler single-stepping on
